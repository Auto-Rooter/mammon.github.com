RE & OS Internals
  • How Debuggers Work

    Rosenberg, 1996: A decent beginning-level discussion of how a debugger works. A rather light treatment, however; not suitable for debugger writers or anti-debugger coders.

  • Indispensable PC Hardware Book

    Messmer, 1997: Hands-down the best buy for x86 hardware info. Thick, heavy, tiny typeface; contains not only 1000+ pages of details hardware info, but 14 appendices covering everything the DOS or system-level assembly programmer will need to know.

  • Panic! Unix Crash Dump Analysis

    Drake & Brown, 1995: Discusses crash dump analysis on Solaris systems -- good not only for recovery and tools usage, but also for info on how to get a look at what is happening in a Solaris system.

  • Reverse Compilation Techniques

    Cifuentes, 1994: A must-read for reverse engineering. The bulk of the thesis is concerned with writing a decompiler, and the dcc program is offered as an example; in particular, however, chapters 3,4,5, and 7 provide excellent theoretical and practical information for reverse engineering.

  • The Undocumented PC

    Van Gilluwe, 1996: The information in this guide is largely superceded by Ralph Brown's Interrup List; this book is concerned with undocumented aspects of PC hardware -- ports, BIOS functions, I/O ranges, etc.

  • Undocumented Windows

    Schulman, Maxey & Pietrek, 1992: A very out-of-date book on the internals of Windows 3.1. Useful for reverse engineering Win16 code, or as a demonstration of how Windows is put together.

  • Unix Internals

    Pate, 1996: This book is an excellent guide to SysV unix; it uses crash and adb to provide an under-the-hood tour of the SCO kernel subsystems. Very highly recommended for Unix engineers.

  • Windows 95 System Programming Secrets

    Pietrek, 1996: Intended for the Win95 developer; this is essentially a reference on debugging and writing utilities for Win32 systems. Slightly out of date with Win2K's release; however this is still a good look at the 'structure' behind Windows products.

  • Applied PC Interfacing, Graphics, and Interrupts

    Buchanan, 1996: Contains asm and C code for interfacing with hardware peripherals. Concise, detailed, inexpensive; biased towards DOS coding.

  • Art of Assembly

    Hyde, 1996: Often referred to as the 'definitive work in the field'; this book offers a thorough and advanced course in x86 assembly language programming. Well worth reading, especially the chapters on Variables and Data Structure, Arithmetic and Logical Operations, Control Structures, Procedures and Functions, and the entire Intermediate and Advanced sections.

  • C for Engineers

    Bramer & Bramer, 1993: Excellent C course for those who want every question about the language covered. Begins with a basic discussion of system architecture and how source code translates to binary machine instructions, then dives in to very clear, very concise, very sound C programming. An excellent way to learn C.

  • Expert C Programming

    Van der Linden, 1994: Written by one of the Sun compiler engineers. In addition to tricks for the C programmer, provides a detailed understanding of what the C code is actually doing. Very highly recommended.

  • Giant Black Book of Computer Viruses

    Ludwig, 1998: Understanding virus coding will provide a solid grasp of assembly language as well. In addition, recovering virii from infected executables is standard work for engineers.

  • Master Class Assembly Language

    Wrox Press, 1995: Introductory text with topics in protected mode programming, code compression and optimization, disassembly, and other advanced asm topics. Too light of a treatment for the expert asm coder, but provides a good introduction to heavyweight topics for the less experienced.

  • PC Underground

    Abacus Press, 1995: Lightweight tome, similar to Master Class Assembly; leans towards games programming, but does provide an assembly language primer and some [weak] protection tips. A C programmer with no asm experience may find this useful, but definitely try-before-you-buy.

  • The 8088 and 8086 Microprocessors

    Triebel & Singh, 2000: An excellent book providing in-depth coverage of x86 programming. Topics range from programming using MASM and debug.exe to accessing peripherals and building circuits. Has brief coverage of processors up to the P-II.

  • The Essence of MicroProcessor Engineering

    Katzen, 1998: Thin, inexpensive book covering microprocessor theory and development using the 68000 as an example. Very good for processor design theory/internals.

  • Windows Assembly Language and Systems Programming

    Kauler, 1997: A must for working with Win32 assembly language. Discusses quirks of the OS [Win16, Win95, DPMI] and demonstrates the typical structure of a disassembled Windows program in its MASM examples. Also introduces the use of OOP features in TASM, much more effectively than the Borland manual.
