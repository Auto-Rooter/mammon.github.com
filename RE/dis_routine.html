Subroutine Recovery
Subroutine recovery, being the translation of a routine from the target into a higher-level language, suitable for integration with an existing project. Distinguised from source code recovery in that the original language is not a concern.


Perhaps the majority of reverse engineering as it is practiced is an attempt to determine the operation of a specific routine or set of routines within the target program. Such attempts might be made to determine how a device driver accesses hardware, how an encryption routine validates or decrypts a signature, or how a program checks for the presence of a cdrom.

The goal in each of these cases is the understanding of a particular routine or function within the target -- not the recovery of the routine, or an understanding of the target as a whole. As such, the target function can be identified using a debugger or other runtime monitoring tools, then isolated from the executable and studied independently.
